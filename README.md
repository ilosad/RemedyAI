📱 Remedy AI
AI 기반 응급 상황 판독 및 즉시 행동 유도 애플리케이션

1. 프로젝트 개요

Remedy AI는 치료(Treatment)에 인공지능(AI)을 결합하여,
예고 없이 발생하는 응급 상황에서 사용자가 신속하고 정확한 판단을 내릴 수 있도록 돕는 Android 애플리케이션입니다.

본 프로젝트의 핵심 목표는 단순히 증상에 대한 정보를 나열하는 것이 아니라,
AI가 사용자의 상태를 **판독(Diagnosis-like 판단)**하고
그 결과를 즉각적인 행동으로 연결하는 것입니다.

즉, Remedy AI는 “의학 정보를 제공하는 앱”이 아니라
**“응급 상황에서 사용자의 의사결정을 보조하는 AI 도구”**를 지향합니다.

2. 문제 인식 및 기획 배경

일상 속에서 갑작스럽게 발생하는 신체 이상이나 통증 상황에서
대부분의 사용자는 다음과 같은 공통된 어려움을 겪습니다.

현재 상태가 응급 상황인지 아닌지 판단하기 어려움

병원을 가야 할지, 기다려도 되는지 결정 기준이 없음

긴 설명이나 전문 용어로 인해 중요한 결론을 놓치기 쉬움

판단을 미루는 동안 시간만 지체되는 문제

특히 응급 상황에서는 “정보의 양”보다 “판단의 명확성”이 훨씬 중요하다고 판단하였습니다.

이러한 문제의식에서 출발하여 Remedy AI는 설명 중심의 의료 정보 제공 방식에서 벗어나,
AI가 상황을 종합적으로 판단하고 사용자에게는 결론과 행동 지침만 전달하는 구조를 목표로 기획되었습니다.

3. 대상 사용자 및 활용 시나리오

Remedy AI는 다음과 같은 사용자를 주요 대상으로 합니다.

갑작스러운 통증이나 이상 증상을 느낀 일반 사용자

응급 상황 판단이 필요한 보호자

평소 고위험군 질환을 보유한 사용자

병원 방문 여부를 빠르게 결정해야 하는 상황

사용자는 의학적 지식이 없어도
문진에 응답하는 것만으로 현재 상황에 대한 AI 기반 판독 결과를 받을 수 있습니다.

4. 핵심 기능 요약

단계별 증상 문진 UI 제공

문진 결과 기반 AI 위험도 판독

응급 / 주의 / 안정 단계 명확화

시각적 요소(아이콘·색상·강조 영역)를 통한 직관적 결과 표시

즉시 해야 할 행동 체크리스트 제공

119 전화 연결 기능

Google Maps API 기반 인근 병원 안내

Firebase Firestore를 활용한 사용자 데이터 저장

5. 사용 API 및 기술 구성

OpenAI GPT API
사용자의 증상, 지속 시간, 통증 강도, 보조 문진 결과를 종합 분석하여
위험도를 내부적으로 판단

Google Maps API
사용자 위치를 기반으로 인근 병원 정보 및 경로 제공

Firebase Firestore
문진 결과 및 AI 판독 기록 저장

6. 전체 시스템 동작 흐름

사용자가 단계별 문진 UI를 통해 증상 정보 입력

입력된 데이터가 GPT API로 전달

AI가 증상 정보를 종합 분석하여 위험도를 판독

판독 결과를 UI 친화적인 결과 형태로 변환

사용자는 즉시 행동 지침, 병원 안내, 전화 연결 기능 사용 가능

이 과정에서
AI의 내부 판단 과정은 사용자에게 직접 노출되지 않으며,
결과 중심의 정보만 제공됩니다.

7. 프로젝트 발전 과정 (4차시 기반)

🔹 1차 구현

앱 기본 구조 설계

문진 UI 구현

AI 연동 오류로 인해 결과 화면 구현에 한계 발생

🔹 2차 구현

AI API 연동 성공

응급도를 점수 형태로 표현

점수만으로는 사용자 이해도가 낮다는 문제 인식

🔹 3차 구현 (중간 발표)

증상 기반 결과 생성

AI가 생성한 긴 텍스트 조언을 화면에 그대로 표시

응급 상황에서 긴 텍스트를 읽기 어렵고,
“지금 무엇을 해야 하는지”가 불명확하다는 피드백 수집

🔹 4차 최종 구현 (최신 버전)

AI의 역할을 설명자 → 판독자로 재정의

AI 결과를 판독 결과 형태로 재구성

응급 등급을 응급 / 주의 / 안정 단계로 명확히 구분

즉시 행동 체크리스트 및 119 전화 연결 추가

문진 문항 수 확장 및 진행바 추가로 신뢰도 향상

UI 구조 개선을 통해 사용자 가독성과 안정감 강화

앱 전반의 로딩 및 응답 속도 개선

이 과정에서
수차례 오류 수정, UI 구조 변경, 피드백 반영을 반복하며
실제 사용 상황을 고려한 설계를 지속적으로 개선하였습니다.

8. AI 활용 방식의 핵심 차별성

기존 버전에서는
AI가 증상에 대해 설명 위주의 긴 텍스트 응답을 제공했습니다.

최신 버전에서는
AI가 내부적으로는 동일하게 복합 분석을 수행하지만,
사용자에게는 이를 판독 결과와 행동 지침으로 요약하여 제공합니다.

즉, 이전: AI가 정보를 설명

현재: AI가 상황을 판독하고, 사용자는 즉시 행동

이는 AI를 단순한 정보 생성 도구가 아닌
의사결정 보조 시스템으로 활용한 구조적 개선입니다.

9. 속도 및 UX 개선

응급 상황에서 가장 중요한 요소 중 하나는 속도입니다.

이전 구현:
AI 응답 전체 로딩 후 화면 표시 → 체감 지연 발생

최신 구현:
UI 즉시 표시 + AI 판단 백그라운드 처리 → 체감 속도 개선

이를 통해
응급 앱으로서의 실질적인 사용성을 크게 향상시켰습니다.
